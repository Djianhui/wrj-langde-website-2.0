## 项目概述

本项目是一个基于Vue 3和Vite构建的现代化前端应用，采用模块化架构设计，包含完整的路由系统、状态管理和国际化支持。项目使用ES模块规范，通过Vite进行快速开发和高效构建。

## 开发环境配置

### 依赖管理

项目使用npm作为包管理工具，核心依赖包括：
- **Vue 3.3.8**：现代JavaScript框架
- **Vite 5.0.0**：快速构建工具
- **Pinia 2.1.7**：状态管理库
- **Vue Router 4.2.5**：路由管理
- **Express 4.18.2**：后端服务框架

### 开发服务器配置

开发环境通过Vite内置服务器提供热更新功能：

```javascript
// 开发服务器配置
server: {
  host: '0.0.0.0',           // 监听所有网络接口
  port: 3001,               // 使用3001端口
  strictPort: true,         // 端口冲突时直接报错
  proxy: {
    '/api': {
      target: 'http://www.langdetech.cn',  // 后端服务地址
      changeOrigin: true,                  // 修改请求源
      rewrite: (path) => path.replace(/^\/api/, '')  // 路径重写
    }
  }
}
```

## Vite构建配置详解

### 基础路径配置

```javascript
// 基础路径配置
base: '/'
```

`base`配置定义了应用的基础路径，影响：
- 静态资源的加载路径
- 路由的相对路径
- API请求的基础URL

### 构建输出配置

```javascript
// 构建配置
build: {
  outDir: 'dist',              // 输出目录
  assetsDir: 'assets',         // 静态资源目录
  sourcemap: false,           // 生产环境禁用source map
  chunkSizeWarningLimit: 1500, // 分块大小警告阈值
  rollupOptions: {
    output: {
      manualChunks(id) {
        if (id.includes('node_modules')) {
          return 'vendor';     // 第三方库单独打包
        }
      }
    }
  }
}
```

### 路径别名配置

```javascript
// 路径别名
resolve: {
  alias: {
    '@': fileURLToPath(new URL('./src', import.meta.url))
  }
}
```

这种配置允许使用`@/components/Example.vue`这样的简短路径引用组件。

## 构建脚本说明

### npm脚本解析

项目提供了以下构建脚本：

```json
{
  "scripts": {
    "dev": "vite",                    // 启动开发服务器
    "build": "vite build",           // 构建生产包
    "preview": "vite preview",       // 预览构建结果
    "server": "node server.cjs",     // 启动后端服务
    "dev:all": "concurrently \"npm run dev\" \"npm run server\""  // 同时启动前后端
  }
}
```

### 开发模式启动

1. **本地开发**：运行`npm run dev`启动Vite开发服务器
   - 访问地址：`http://localhost:3001`
   - 支持热更新(HMR)
   - 自动代理API请求到后端服务

2. **全栈开发**：运行`npm run dev:all`
   - 同时启动前端开发服务器和后端服务
   - 方便调试前后端交互

### 生产构建

执行`npm run build`生成生产环境代码：
- 将源码编译为优化的JavaScript
- 生成静态HTML文件
- 打包CSS和图片资源
- 创建source map用于调试

### 构建预览

运行`npm run preview`启动本地预览服务器：
- 模拟生产环境行为
- 测试构建产物的正确性
- 验证静态资源加载

## 构建产物分析

### 目录结构

构建完成后，`dist`目录包含以下内容：

```
dist/
├── index.html          # 主入口HTML文件
├── assets/             # 静态资源目录
│   ├── main.[hash].js  # 主应用代码
│   ├── vendor.[hash].js # 第三方库
│   ├── images/         # 图片资源
│   └── styles/         # 样式文件
└── [其他静态资源]      # 如字体、图标等
```

### 静态资源特性

1. **文件名哈希**：所有静态资源文件名包含哈希值，确保浏览器缓存策略
2. **按需加载**：支持路由级别的代码分割
3. **资源压缩**：JavaScript和CSS经过压缩优化
4. **图片优化**：自动优化图片格式和尺寸

### HTML模板

主HTML文件包含以下关键元素：

```html

  <meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## Nginx部署配置

### 基础配置

```nginx
server {
    listen 80;
    server_name langdetech.cn www.langdetech.cn;
    root /var/www/langdetech/dist;
    index index.html;
}
```

### Gzip压缩配置

```nginx
# 开启gzip压缩
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
gzip_min_length 1k;
gzip_vary on;
gzip_proxied any;
```

### 静态资源缓存策略

```nginx
# 静态资源缓存设置
location /assets/ {
    expires 7d;
    add_header Cache-Control "public, no-transform";
}
```

### 单页应用路由回退

```nginx
location / {
    try_files $uri $uri/ /index.html;
    add_header Cache-Control "no-cache, must-revalidate";
}
```

这个配置确保所有未匹配的URL都返回`index.html`，支持Vue Router的客户端路由。

### API反向代理

```nginx
# API 代理配置
location /api/ {
    proxy_pass http://localhost:3001/;  # 后端服务地址
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_cache_bypass $http_upgrade;
}
```

### 错误页面配置

```nginx
# 错误页面配置
error_page 404 /index.html;
error_page 500 502 503 504 /50x.html;
location = /50x.html {
    root /usr/share/nginx/html;
}
```

## Docker化部署方案

### Dockerfile示例

```dockerfile
FROM nginx:alpine
COPY dist/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### Docker Compose配置

```yaml
version: '3.8'
services:
  frontend:
    build: .
    ports:
      - "80:80"
    volumes:
      - ./logs:/var/log/nginx
    restart: unless-stopped
  
  backend:
    image: node:18-alpine
    working_dir: /app
    command: npm run server
    ports:
      - "3001:3001"
    volumes:
      - .:/app
    depends_on:
      - database
    restart: unless-stopped
```

### 容器化部署流程

1. **构建镜像**：`docker build -t lande-frontend .`
2. **运行容器**：`docker run -d -p 80:80 --name frontend lande-frontend`
3. **集群部署**：使用Kubernetes或Docker Swarm管理容器

## 常见部署问题排查

### 404路由回退问题

**症状**：访问SPA路由时出现404错误

**原因**：Nginx未正确配置单页应用路由回退

**解决方案**：
```nginx
# 确保location配置正确
location / {
    try_files $uri $uri/ /index.html;
}
```

### CORS跨域问题

**症状**：API请求被浏览器阻止

**原因**：前后端域名不一致或CORS配置缺失

**解决方案**：
1. 在后端服务中添加CORS中间件
2. 或者在Nginx中配置代理头

### 静态资源加载失败

**症状**：CSS、JS文件无法加载

**原因**：基础路径配置错误或文件权限问题

**解决方案**：
1. 检查`vite.config.js`中的`base`配置
2. 确认Nginx root路径指向正确的dist目录
3. 检查文件权限：`chmod -R 755 dist/`

### 缓存问题

**症状**：更新后页面显示旧版本

**原因**：浏览器缓存了旧的静态资源

**解决方案**：
1. 使用文件名哈希：`[name].[hash].[ext]`
2. 设置适当的缓存头
3. 强制刷新浏览器缓存
